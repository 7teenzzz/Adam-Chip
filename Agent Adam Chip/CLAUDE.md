# Agent Adam Chip — правила редактирования персоны

## Порядок загрузки

Файлы загружаются в системный промпт в этом порядке (из `Config.json agent.persona_paths`):

1. `About/System.md` — системные ограничения и режимы работы Адама
2. `About/Identity.md` — характер, голос, самоощущение персонажа
3. `About/Lore.md` — история, происхождение, контекст инсталляции
4. `About/Abilities.md` — возможности: что Адам умеет и чего нет

Порядок не случайный — он определяет приоритет инструкций в промпте. Менять порядок только через `Config.json agent.persona_paths` одновременно с самими файлами.

## Запреты

- **Никакого JSON, code blocks (```), markdown-таблиц** — LLM получает plain text из этих файлов напрямую в системный промпт. Любая разметка окажется в тексте ответа Адама.
- Заголовки (`##`) влияют на парсинг в `System/adam/prompt.py` — проверять после изменений структуры
- Язык файлов: только **русский**

## Редактирование

- `Identity.md` и `Lore.md` — горячий путь: изменения немедленно влияют на характер ответов
- `System.md` содержит хард-ограничения — изменять осторожно, тестировать после правок
- `Abilities.md` — описание возможностей; несоответствие с реальным поведением сбивает пользователей
- После правок проверить: файл не содержит markdown-разметки кроме заголовков `##` и обычного текста
