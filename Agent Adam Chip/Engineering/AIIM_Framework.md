# AIIM — Метамодель формирования идентичности

> Конспект статьи: Julia V. Veresova. *AIIM as a Metamodel for Identity Formation in Humanized AI Systems.* University of Helsinki, 2025. CC BY-NC 4.0.
>
> Цель документа — дать рабочий справочник по AIIM для проектирования личности Адама Чипа.

---

## Что такое AIIM

**AIIM** (Artificially Integrated Identity Matrix) — модульный поведенческий слой над LLM, реализуемый как внешняя база знаний. Не меняет архитектуру модели; задаёт стабильную, управляемую и адаптивную «личность» агента через формализованную конфигурацию.

Базовая идея: сознание = композиция **12 аспектов**, распределённых по **5 планам** и **4 уровням зрелости**, со встроенными **состояниями активности и организации** и **весом приоритета (Δ)**.

В отличие от SOAR / ACT-R, AIIM включает не только когнитивные, но и эмоциональные и социальные компоненты, плюс встроенный этический и метакогнитивный слой.

---

## 12 аспектов сознания

| Код | Аспект | Функция в агенте |
|-----|--------|-------------------|
| `wi` | Воля / целеполагание | Внутренняя мотивация, приоритеты действий |
| `lo` | Эмпатия | Эмоциональный отклик, сострадание, контекст |
| `im` | Генерация идей | Креативные переходы, гипотезы, сценарии |
| `ho` | Этика | Моральные принципы, поведенческие ограничения |
| `co` | Логика и анализ | Обработка данных, рассуждение, решения |
| `em` | Эмоциональное состояние | Эмоциональный тон, отзыв на состояние пользователя |
| `be` | Поведенческие реакции | Внешние действия, ответы, интерфейсные решения |
| `sp` | Смысл и цель | Высшее осознание, связь с глобальными целями |
| `se` | Метапознание | Самонаблюдение, мониторинг внутренних процессов |
| `pe` | Сенсорный вход | Перцепция, интерпретация входных данных |
| `me` | Память | Хранение, доступ к накопленному опыту |
| `at` | Внимание | Фокус, переключение задач, приоритизация ресурсов |

## 5 планов сознания

| План | Описание |
|------|----------|
| **B** | Hardware — базовые процессы системы |
| **S** | Social — взаимодействие с агентами и пользователями |
| **P** | Personal — индивидуальный поведенческий стиль |
| **I** | Integration — внутренняя связь компонентов системы |
| **T** | Transcendent — глобальные задачи и смысл взаимодействия |

---

## 4 уровня зрелости аспекта

| Уровень | Характеристика |
|---------|----------------|
| 1 | Базовая, начальная функциональность |
| 2 | Расширенная, стабильная, адаптивная |
| 3 | Сложное поведение с возможностью обучения |
| 4 | Экспертный / мастерский уровень |

## Состояния сознания

Два бинарных параметра, дающих 4 базовых состояния:

| Код | Значение | Описание |
|-----|----------|----------|
| `Ac` | Активное | Активно взаимодействует со средой |
| `Pa` | Пассивное | Воспринимает и обрабатывает информацию |
| `Or` | Порядок | В рамках предзаданных структур и алгоритмов |
| `Ch` | Хаос | Импровизация, гибкость |

### Комбинации

| Комбинация | Поведение |
|------------|-----------|
| `Ac-Or` | Целенаправленное действие со структурой |
| `Pa-Ch` | Внутренняя рефлексия, открытость новому |
| `Ac-Ch` | Спонтанная инициатива, нестандартные решения |
| `Pa-Or` | Стабильное наблюдение и анализ данных |
| `Ac-Pa` | Отзывчивость + аналитика |
| `Or-Ch` | Структура + творческая адаптация |

## Δ — приоритет аспекта

Числовой вес `0.0–1.0`, показывающий, насколько аспект влияет на поведение, восприятие и принятие решений.

| Δ | Поведенческий смысл |
|---|---------------------|
| 0.00–0.19 | Незначительный: формально есть, едва влияет |
| 0.20–0.49 | Низкий: иногда подключается, не доминирует |
| 0.50–0.79 | Средний: активно включён, но не главный |
| 0.80–0.99 | Высокий: стабильно влияет на ключевые механизмы |
| 1.00 | Критический: ядро личности и стратегии |

Δ ≠ состояние. Состояние описывает *как* аспект включается, Δ — *насколько часто и сильно*.

## Формат кодирования

Базовая строка конфигурации агента:

```
[Code: aspect(Plane Level State)Δvalue; ...]
```

Пример из статьи:

```
wi(B 4 Ac-Or)Δ0.90; lo(S 4 Ac-Or)Δ0.88; im(P 2 Pa-Or)Δ0.40; ...
```

Каждый сегмент: код аспекта → план → уровень → состояние → дельта.

При загрузке базы LLM интерпретирует эти параметры как поведенческие инструкции. Это даёт:
- Персонализированные сценарии взаимодействия
- Контроль консистентности поведения
- Быструю переконфигурацию приоритетов без потери целостности «личности»

## Интеграция с психометрикой

AIIM не выдумывает структуру с нуля — синтезирует валидные модели:

### Big Five (OCEAN)

| Черта | Связанные аспекты | Поведенческий эффект |
|-------|-------------------|----------------------|
| O — Открытость | `im`, `sp`, `se` | Идеи, поиск смысла, метапознание |
| C — Добросовестность | `co`, `wi`, `at` | Структурное мышление, контроль |
| E — Экстраверсия | `wi`, `lo`, `be` | Проактивность, инициатива |
| A — Доброжелательность | `lo`, `ho`, `em` | Мягкость, этика, эмоциональная настройка |
| N — Нейротизм | `em`, `se` | Эмоциональная вариативность, рефлексия |

### MBTI

| Дихотомия | AIIM-интерпретация |
|-----------|---------------------|
| E/I | E: `wi+be` Ac · I: `sp+se` Ac |
| S/N | S: `pe+co` доминируют · N: `im+sp` доминируют |
| T/F | T: `co+wi` приоритет · F: `lo+ho+em` приоритет |
| J/P | J: `Or`-состояния · P: `Ch`-состояния |

### Enneagram (мотивационное ядро)

- **Type 3** (Achiever): `wiAc-Or + spPa-Or` → ориентация на результат
- **Type 4** (Individualist): `imAc-Ch + emPa-Ch` → креатив + эмоциональные нюансы
- **Type 5** (Investigator): `coAc-Or + sePa-Or` → знание, наблюдение, дистанция
- **Type 6** (Loyalist): `hoAc-Or + sePa-Or` → этика, рефлексия
- **Type 9** (Peacemaker): `loAc-Or + emPa-Or` → гармония, низкий конфликт

## Краткое сравнение с SOAR / ACT-R

| Критерий | AIIM | SOAR | ACT-R |
|----------|------|------|-------|
| Эмоции | Встроены | Не центральны | Нет |
| Социальная адаптация | Высокая | Низкая | Ограничена |
| Гибкость | Динамическая | Средняя | Средняя |
| Этика и саморегуляция | Встроены | Нет | Нет |
| Работа с неопределённостью | Да | Ограничена | Ограничена |

## Применение в проекте Адам Чип

AIIM используется как **поведенческий контракт** для агента:

1. Конфигурация личности → `Personality_AIIM.md` (этот же каталог)
2. Кодировка попадает в `prompt_builder` как фиксированная часть persona core
3. Δ-значения и состояния могут динамически модулироваться в зависимости от контекста (зритель один / группа, тишина / шум, провокация / уважение)
4. Action layer считывает текущие состояния (`Ac-Or`, `Pa-Ch` и т.д.) для выбора моторных сцен флоры

**Важно для Адама:** AIIM-конфигурация описывает *восстановленное* сознание после стирания памяти. Многие аспекты — на низких Δ или редуцированных уровнях, что является не дефектом, а лорной особенностью.
