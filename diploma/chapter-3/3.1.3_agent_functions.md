# 3.1.3 Функции нейроагента

В данном разделе систематизированы пять ключевых функций нейроагента «Адам Чип» в соответствии с их ролью в поведенческом цикле: восприятие, интерпретация, речевой отклик, исполнительные команды и удержание линии во времени. Каждая функция сопоставлена с реализующим её программным компонентом.

## Функция 1: Восприятие

Восприятие охватывает получение сенсорных данных из физической среды. Реализовано двумя каналами: звуковым и визуальным. Звуковой канал включает систему обнаружения ключевого слова (OpenWakeWord, модель `data/wake_word/adam.onnx`), детектор голосовой активности (WebRTC VAD, реализован в `System/adam/webrtc_vad.py`) и систему распознавания речи (WhisperX ASR, Docker-сервис на порту 8095, модель `small`, язык `ru`). Визуальный канал основан на захвате изображений камерой OV5640 через ESP32 MJPEG-поток (`media.video.esp_mjpeg_url: http://192.168.0.171:81/stream`) и последующем анализе языковой моделью зрения VILA 1.5-3b (порт 8084).

Восприятие обеспечивает непрерывный мультимодальный поток: звук — для диалога, изображение — для осознания социального контекста сцены.

## Функция 2: Интерпретация

Интерпретация выполняется языковой моделью Gemma 4 E4B (квантизация Q4_K_XL, формат GGUF) через сервис llama.cpp (порт 8081, OpenAI-совместимый API). Модель получает структурированный промпт, собранный `PromptBuilder` (`System/adam/prompt.py`): идентификационные файлы персонажа, параметры из `Tuning.json`, релевантные эпизоды памяти, описание текущей сцены и транскрипцию реплики посетителя.

Ключевой инвариант интерпретации: языковая модель возвращает исключительно чистый русский текст без разметки, JSON или code blocks. Извлечение state markers для командного контура осуществляется action layer отдельно и после генерации (см. раздел 3.2.6). Ограничение `services.llm.max_tokens: 40` удерживает ответы в рамках TTS-бюджета латентности.

## Функция 3: Речевой отклик

Речевой отклик реализован через синтез речи Silero TTS v5_5_ru (сервис на порту 8082, голос `eugene`). Синтез запускается параллельно с командным контуром: пока TTS генерирует и воспроизводит ответ через HDMI-выход (`plughw:1,3`), MCU-клиент отправляет команды моторному слою. Задержка до TTS снижается механизмом фраз-заполнителей: при превышении `tts.filler_delay_ms: 800` мс воспроизводится «Хм…» до готовности ответа языковой модели.

В режиме half-duplex (`safety.half_duplex_mute: true`) микрофон заглушается на весь период воспроизведения TTS, исключая акустическую обратную связь.

## Функция 4: Исполнительные команды

Исполнительные команды обеспечивают управление физическим телом инсталляции — моторным слоем на базе ESP32-S3. Функция реализована связкой `ActionLayer` (`System/adam/action.py`) и `MCUClient` (`System/adam/device.py`). `ActionLayer` анализирует текст ответа языковой модели на наличие state markers — тегов вида `[название_сцены]` — и сверяет каждую команду с whitelist допустимых сцен (`mcu.allowed_scenes`). Только прошедшие валидацию команды передаются в `MCUClient`, который отправляет HTTP-запросы к ESP32 по адресу `http://192.168.0.171`. Команды, не прошедшие валидацию, преобразуются в `no_action`, но голосовой ответ при этом не прерывается.

Данная нормативная редукция обеспечивает физическую безопасность и предсказуемость моторного поведения (см. раздел 3.2.6 для детального описания).

## Функция 5: Удержание линии во времени

Удержание линии во времени реализует темпоральную связность агента: сохранение событий прошлого и их использование для формирования контекстно-последовательного поведения. Реализуется через многоуровневую систему памяти: `SessionAccumulator` (`System/adam/episodic.py`, 23 рёбра) накапливает реплики текущей сессии; `EpisodicMemory` (`System/adam/memory.py`, 29 рёбер) хранит эпизоды в SQLite (`data/adam/memory.sqlite3`) и предоставляет механизм выборки по релевантности (salience scoring); `Engineering/consolidator.py` (cron) ежедневно консолидирует события в summaries (`data/adam/summaries/`) и notes (`data/adam/notes/`). Биографическая рамка персонажа (`Agent Adam Chip/About/*.md`) обеспечивает постоянный нарратив поверх эпизодических данных.

Таким образом, агент не только помнит текущий диалог, но поддерживает нарративную идентичность через смену сессий и дней.

## Соответствие критериям квазисубъектности

Пять описанных функций обеспечивают реализацию шести из восьми критериев квазисубъектности (раздел 2.1): восприятие → воплощённость (2.1.7) и интеракционность (2.1.6); интерпретация → тип агентности (2.1.2); речевой отклик + исполнение → воплощённость; удержание во времени → темпоральная связность (2.1.5) и устойчивость идентичности (2.1.3).
