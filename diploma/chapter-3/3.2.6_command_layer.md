# 3.2.6 Командный контур

В данном разделе описывается командный контур нейроагентной системы — механизм трансляции языкового ответа в физические действия моторного слоя ESP32. Рассматриваются нормативная редукция как архитектурный принцип, компоненты контура и обеспечение безопасности.

## Нормативная редукция как принцип

Ключевым архитектурным принципом командного контура является **нормативная редукция**: сложный и вариативный языковой вывод языковой модели редуцируется к фиксированному словарю допустимых команд перед отправкой на физическое устройство. Такой подход обеспечивает предсказуемость и безопасность моторного поведения при сохранении вербальной вариативности агента.

Цепочка редукции: языковая модель генерирует свободный текст → `ActionLayer` извлекает state markers → сверяется с whitelist → прошедшие валидацию команды → `MCUClient` → HTTP → ESP32. Команды, не прошедшие валидацию, преобразуются в `no_action`; голосовой ответ при этом продолжается без прерывания (инвариант: «action failure ≠ silence»).

## Компоненты командного контура

**`ActionLayer`** (`System/adam/action.py`) — слой валидации и трансляции. Анализирует текст ответа языковой модели на наличие тегов вида `[название_сцены]`. Сверяет каждый извлечённый тег с белым списком допустимых сцен из конфигурации:

```json
"mcu": {
  "allowed_scenes": ["boot_idle", "all_on", "alternating"],
  "channels": { "min": 0, "max": 15, "value_min": 0, "value_max": 4095 }
}
```

Параметры безопасности из раздела `safety`:
- `motor_max_duration_ms: 2500` — жёсткий лимит длительности активации моторов
- `motor_default_duration_ms: 900` — длительность по умолчанию для команд без явной
- `motor_cooldown_ms: 250` — минимальный интервал между последовательными командами

**`MCUClient`** (`System/adam/device.py`, 25 рёбер в граф-отчёте) — HTTP-клиент для взаимодействия с ESP32-S3 по статическому IP `192.168.0.171`. Таймаут запроса `mcu.timeout_sec: 1` — намеренно короткий для fire-and-forget семантики. Команды не блокируют голосовой поток.

## Взаимодействие с языковой моделью

Языковая модель работает с ключевым ограничением: она не знает о физических командах явно. Системный промпт содержит нормативную инструкцию о допустимых тегах состояния, но языковая модель производит тег как часть натурального текстового ответа. `ActionLayer` выступает «переводчиком» между языковым и физическим уровнями.

Этот дизайн соответствует **ограниченно-внутренней нормативности** (второй тип по критерию 2.1.4): правила действия заданы извне (whitelist в Config.json), но система адаптирует выбор тега к контексту реплики через языковую модель.

## Параллельность TTS и MCU

Голосовой ответ и моторные команды выполняются **параллельно**: TTS-воспроизведение и отправка HTTP-команды MCU инициируются одновременно после генерации ответа языковой модели. Это обеспечивает синхронность «тела» (light/sound/vibration через ESP32) и «голоса» агента — важнейшее условие создания ощущения присутствия в выставочном контексте.

## Нормативность и безопасность (критерий 2.1.4)

Командный контур является основным воплощением нормативности системы. Жёсткая валидация через whitelist, ограничения длительности и cooldown предотвращают механическое повреждение инсталляции и непредсказуемое поведение в публичном пространстве. Такая архитектура соответствует уровню «ограниченно-внутренней нормативности» (критерий 2.1.4, раздел 3.4.2): внешние ограничения обеспечивают предсказуемость; языковая модель сохраняет адаптивность в пределах этих рамок.
