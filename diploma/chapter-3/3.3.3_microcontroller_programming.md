# 3.3.3 Программирование микроконтроллера

В данном разделе описывается firmware ESP32-S3, управляющая периферийным слоем инсталляции «Адам Чип»: состав функциональных блоков, организация HTTP API, механизмы безопасности и конфигурация сети. Прошивка реализована в среде PlatformIO и размещена в директории `Subsystem/AdamsServer/`.

## Архитектура firmware

Центральным архитектурным элементом прошивки является `RuntimeState` — глобальная структура, содержащая текущее состояние всех подсистем: сетевое подключение, состояние I2S-трактов, активную сцену PCA9685, метрики видеолатентности и диагностику загрузки. Доступ к структуре из разных задач FreeRTOS защищён через `portMUX_TYPE` — аппаратный спинлок ESP32-S3, обеспечивающий атомарность операций чтения-записи без использования RTOS-примитивов. Это позволяет любому модулю опросить или обновить глобальное состояние системы за минимальное время блокировки.

Вторым сквозным элементом является `BootDiagnostics` — модуль пошаговой диагностики запуска. Функция `bootLogf()` является наиболее связанным узлом всего кодового графа прошивки (32 ребра): она вызывается из всех шести подсистем — сети, аудио, камеры, PCA9685, сенсоров и веб-сервера — что делает последовательность загрузки прозрачной и верифицируемой. Анализ serial-лога (`artifacts/_serial_log.txt`) подтверждает десятишаговую последовательность: sensors → ethernet → ota → sounds → camera → mic → speaker → pca9685 → web → running, завершающуюся примерно за 3 секунды с воспроизведением boot sound на 5,8 секунде.

Firmware ESP32-S3 организована в три функциональных блока:

**Communication Block (коммуникационный блок)** реализует два независимых HTTP-сервера:
- Основной сервер (порт 80) — принимает команды от оркестратора Jetson. REST API охватывает управление сценами, отдельными PWM-каналами, сенсорный polling и системную диагностику. WebSocket на том же порту обеспечивает push-телеметрию.
- Аудиосервер (порт 81) — принимает PCM-данные для воспроизведения через PCM5102A. Разделение на два сервера обусловлено ограничением стека lwIP на ESP32: максимальное число одновременных сокетов (`CONFIG_LWIP_MAX_SOCKETS: 16`) делает критически важным минимизацию конкуренции за сокеты между командным и аудиопотоками.

**Motor Block (моторный блок)** управляет тремя каналами физического воздействия:
- *Световой канал*: управление PCA9685 через I²C; 16 каналов 12-bit PWM (0–4095); отдельные функции для установки конкретного канала, воспроизведения сцены (preset pattern), плавного нарастания/спада.
- *Звуковой канал*: потоковое воспроизведение WAV/PCM через PCM5102A и PAM8403; буферизация для снижения задержки.
- *Вибрационный канал*: прямое GPIO-управление вибромоторами; поддержка паттернов активации с настраиваемой длительностью.

**Sensor Block (сенсорный блок)** предоставляет данные внешних сенсоров по запросу:
- OV5640: MJPEG-поток по отдельному URL (`/stream`, порт 81)
- INMP441×2: PCM-аудиоклипы по `/api/audio/clip`
- TEMT6000: значение освещённости
- BTE16-19: биометрические данные

## Сетевая конфигурация

Ethernet-подключение через W5500 SPI обеспечивает стабильную проводную сеть в условиях выставочного зала, где Wi-Fi может быть перегружен. Статический IP `192.168.0.171` задан на уровне firmware. Использование SPI Ethernet вместо встроенного Wi-Fi ESP32 обусловлено требованием надёжности: потеря Wi-Fi-соединения в mid-turn неприемлема в публичном пространстве.

## Механизмы безопасности и устойчивости

Прошивка содержит три уровня защиты от нештатных ситуаций.

**Idle Scene Fallback**: при потере связи с Jetson или отсутствии команд в течение timeout ESP32 самостоятельно переходит в защитную сцену `boot_idle` — спокойное фоновое световое поведение без движущихся частей. Сцена хранится в firmware и не требует команды с Jetson для активации.

**Watchdog Timer**: ESP32 аппаратный watchdog перезагружает MCU при зависании задачи.

**Graceful HTTP errors**: все API-эндпоинты возвращают JSON-ошибки при некорректных параметрах вместо паники firmware.

## Инструменты разработки и деплоя

Сборка и прошивка осуществляются из Windows dev-машины через PlatformIO. Для прошивки предусмотрены PowerShell-скрипты: `Subsystem/AdamsServer/tools/flash_com7.ps1` — прошивка через COM7 (UART); `Subsystem/AdamsServer/tools/flash_ota.ps1` — OTA-обновление по Wi-Fi/Ethernet. Порт COM6 используется для чтения логов приложения без прерывания работы.

Чувствительная конфигурация (SSID, пароли, приватные ключи) хранится в `Subsystem/AdamsServer/config/PrivateConfig.h`, который исключён из репозитория через `.gitignore`.
