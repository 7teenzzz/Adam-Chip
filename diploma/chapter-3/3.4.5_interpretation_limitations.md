# 3.4.5 Интерпретация результатов и ограничения

В данном разделе представлены интерпретация наблюдаемых результатов работы нейроагентной инсталляции «Адам Чип», анализ эмерджентных эффектов и документирование архитектурных ограничений. Раздел операционализирует критерий **уровня эмерджентности** (раздел 2.1.8, статус PARTIAL) и формирует основу для определения направлений дальнейшего развития системы.

## Наблюдаемые эмерджентные эффекты

Анализ поведения системы в ходе тестирования и выставочных показов выявил четыре эмерджентных эффекта — паттерны поведения, не запрограммированные явно, но возникающие из взаимодействия модулей.

**Эффект 1: Композиционный персонаж.** «Адам Чип» как воспринимаемый характер возникает только при совместной работе всех модулей. Языковая модель без персона-файлов и Tuning.json производит generic ответы; LLM без памяти теряет нарративную нить; моторный слой без голоса создаёт «безмолвный жест». Ни один компонент по отдельности не воспроизводит эффект присутствия персонажа. Это свидетельствует об интеракционном уровне эмерджентности (критерий 2.1.8): поведение системы не редуцируемо к поведению отдельных модулей.

**Эффект 2: Контекст-зависимая модуляция.** Память, описание сцены и параметры Tuning.json совместно модулируют тональность ответа без явного «переключения режима». Агент может быть более «сдержанным» при первом взаимодействии и более «открытым» с возвращающимся посетителем — без изменения базовых параметров персонажа. Данный эффект возникает из взаимодействия нескольких слоёв промпта и не задан ни одним из них напрямую.

**Эффект 3: Distributed body presence.** Три канала физического воздействия — свет (PCA9685/LED), звук (PCM5102A) и вибрация (GPIO) — синхронно создают ощущение «живого присутствия», недостижимое ни одним каналом по отдельности. Совместная реакция светофлоры, аудиофлоры и виброфлоры на диалоговый turn воспринимается зрителями как «отклик тела», а не «управляемые огни».

**Эффект 4: Latency-driven personality.** Фиксированная задержка языковой модели (~9 секунд) и механизм filler-фраз непреднамеренно формируют «задумчивый» характер персонажа. Пауза перед ответом воспринимается как «размышление», а не как сбой системы. Этот эффект является примером инженерного ограничения, трансформировавшегося в художественное качество.

## Ограничения реализованной системы

**Ограничение 1: Отсутствие спонтанной речи.** Языковая модель не инициирует голосовые высказывания без wake word. Проактивность ограничена уровнем восприятия (SceneWorker, SessionWatcher). Причина: стоимость LLM inference (~9 с, ~5 Вт·ч на turn) на Jetson Orin NX делает нецелесообразным регулярный вызов модели в отсутствие обращения.

**Ограничение 2: Задержка языковой модели.** Gemma 4 E4B использует гибридную схему внимания с SWA (Sliding Window Attention, размер окна 512), что обуславливает полный сброс KV-кэша между turn'ами и full prefill ~2781 токена при каждом обращении. Следствие: total turn latency 11–15 секунд. Возможные пути снижения: уменьшение размера промпта, переход на модель с меньшим KV-кэшем, кэширование prefix-части промпта.

**Ограничение 3: Несоответствие языковой модели проектному заявлению.** В ранней версии диплома в качестве языковой модели указана Cosmos Reasoning 2 2B. В реализованной системе используется `gemma-4-E4B-it-UD-Q4_K_XL`. Замена продиктована практическими соображениями: доступность квантизации Q4_K_XL для aarch64 и подтверждённые показатели ~65 токен/с на Jetson. Поведенческие характеристики системы не зависят от конкретной модели — они определяются архитектурой оркестрации.

**Ограничение 4: AIIM-формула не реализована буквально.** Формальная запись `wi(B 4 Ac-Or)Δ0.90;...` является теоретическим языком описания; практическая конфигурация осуществляется через структурированный JSON (`Tuning.json`). Рефлексивный уровень AIIM (автоматическое изменение уровней зрелости на основе опыта) не реализован.

**Ограничение 5: Одиночный агент.** Система реализует single-agent архитектуру. Мультиагентные эффекты (кооперативная интеракционность, координационное поведение) не наблюдаемы по определению конфигурации.

## Направления дальнейшего развития

На основе анализа ограничений и выявленных эмерджентных эффектов сформулированы приоритетные направления развития системы.

**Рефлексивный уровень AIIM**: реализация автоматического изменения параметров `Tuning.json` на основе паттернов взаимодействия — «обучение» агента на собственном опыте без перепрограммирования.

**Спонтанная речевая инициатива**: разработка idle-scheduler с адаптивным пороговым условием для запуска голосовых реплик при длительном присутствии посетителей без обращения.

**Снижение латентности LLM**: исследование prefix caching для инвариантной части промпта (системная инструкция + персона-файлы) и возможности квантизации меньшей битности.

**Расширение sensory vocabulary**: добавление биометрической обратной связи через BTE16-19 (пульс посетителя) как дополнительного контекстуального канала для языковой модели.

## Обобщающая оценка

Нейроагентная система «Адам Чип» реализует примерно **85% заявленного функционала**: полностью — критерии агентности, идентичности, нормативности, темпоральной связности и воплощённости; частично — автономизацию, интеракционность и эмерджентность. Расхождения между проектными заявлениями и реализацией объяснимы инженерными решениями, продиктованными ограничениями edge-платформы, и не нарушают целостности художественной концепции.
