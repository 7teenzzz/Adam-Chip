# 3.2.4 Память и контекст

В данном разделе описывается многоуровневая система памяти нейроагента «Адам Чип»: архитектура хранилищ, механизм salience-based выборки и процесс консолидации. Система памяти обеспечивает темпоральную связность агента — его способность поддерживать нарративную линию через смену сессий и диалогов.

## Архитектура многоуровневой памяти

Память «Адама Чипа» организована по пяти уровням, каждый из которых реализует отдельный горизонт и гранулярность хранения:

**Уровень 1 — Рабочая память (Working Memory).** Реализована через `SessionAccumulator` (`System/adam/episodic.py`, 23 рёбра). Накапливает реплики и события текущей сессии в оперативной памяти. При завершении сессии содержимое сохраняется в SQLite и JSONL. Жизненный цикл: от старта до завершения сессии.

**Уровень 2 — Суммаризированная память (Summary Memory).** Реализована через директорию `data/adam/summaries/`. Ежедневный consolidator (`Engineering/consolidator.py`, запускается как cron) анализирует JSONL-поток событий и формирует краткие тематические summaries прошедших сессий. Жизненный цикл: дни и недели.

**Уровень 3 — Заметки (Notes).** Директория `data/adam/notes/`. Selectively-written фрагменты: значимые наблюдения, паттерны взаимодействия, тематические акценты. Формируются консолидатором по критериям salience. Жизненный цикл: постоянный до ручной очистки.

**Уровень 4 — Эпизодическая память (Episodic Memory).** Реализована через `EpisodicMemory` (`System/adam/memory.py`, 29 рёбер) с хранилищем `MemoryStore` (SQLite, `data/adam/memory.sqlite3`, 14 рёбер). Структурированные записи об эпизодах с метаданными: время, участник, salience score, ключевые темы. Механизм выборки: семантический поиск по релевантности запросу текущего turn.

**Уровень 5 — Биографическая рамка (Permanent Bio).** Файлы `Agent Adam Chip/About/*.md`: `Identity.md`, `Lore.md`, `Abilities.md`. Постоянный нарратив, не изменяемый в runtime. Обеспечивает базовую идентификацию персонажа независимо от эпизодической истории.

## Механизм salience scoring

Не все события сессии одинаково значимы для долговременной памяти. Salience scoring (`System/adam/episodic.py`) вычисляет оценку значимости для каждого эпизода на основе нескольких факторов: эмоциональной интенсивности реплики, тематической новизны (отсутствие в существующей памяти), длительности диалогового обмена, присутствия проактивных сигналов (сцена, фоновая реакция).

Эпизоды с высоким salience score приоритизируются при выборке для включения в промпт, а также при консолидации — определяют содержимое summaries и notes. Такой подход обеспечивает **информационную компрессию**: агент помнит «важное», а не «всё», что аналогично селективности человеческой долговременной памяти.

## Retrieval-Augmented Generation в диалоговом контексте

Выборка релевантных эпизодов из `EpisodicMemory` перед каждым turn аналогична механизму Retrieval-Augmented Generation (RAG) [46]: перед формированием промпта выполняется запрос к памяти с дескриптором текущей ситуации, возвращающий N наиболее релевантных эпизодов. Они включаются в четвёртый слой промпта (см. раздел 3.2.3), обогащая контекст сведениями о прошлых взаимодействиях без полного включения всей истории.

## Консолидация памяти

`Engineering/consolidator.py` — ежедневный cron-процесс, перерабатывающий сырой JSONL-поток (`data/adam/events.jsonl`) в структурированные summaries и notes. Консолидатор работает по расписанию в off-exhibition время, не конкурируя за ресурсы Jetson в часы выставки.

## Темпоральная связность (критерий 2.1.5)

Описанная архитектура реализует **нарративную темпоральную связность** (третий уровень по шкале критерия 2.1.5): агент соединяет события через сессии через эпизодическую память и summaries, выстраивая нарративную линию. Элементы процессуальной связности присутствуют через consolidator, однако полный рефлексивный цикл (переосмысление прошлого на основе нового опыта) не реализован.
