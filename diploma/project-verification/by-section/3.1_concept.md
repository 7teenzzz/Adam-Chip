# Section 3.1 — Концепция проекта и художественная логика

## 3.1.1 Концептуальная основа

**Verification.** Концепция «цифрового симбионта» воплощена через гибридную аппаратную инсталляцию (Jetson + ESP32 + technoflora). Сбои/задержки честно integrированы:
- Filler phrases (`filler_phrase: "Хм..."`) маскируют LLM latency.
- Power gate отказывается стартовать без MAXN — material constraints явны.
- Half-duplex mute предотвращает hallucinated speech во время TTS.

**Status:** **FULL** — концепт реализован через инженерные ограничения, не через имитацию.

## 3.1.2 Логика поведения

**Verification.** Сочетание реактивного + проактивного режимов:
- Реактивное: wake word → ASR → LLM → TTS + MCU ✅
- Проактивное (perception): SceneWorker, SessionWatcher, EspAudioHealthMonitor ✅
- Проактивное (speech без триггера): ❌ отсутствует

**Status:** **PARTIAL** — proactive perception, но не proactive speech.
