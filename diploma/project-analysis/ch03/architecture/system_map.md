## Центральный orchestrator

**Module:** `Orchestrator.py`

**Theoretical role.** Диспетчер состояний. Принимает сигналы от перцептивных модулей, инициирует сбор контекста, запускает обращение к LLM, передаёт результат в постобработку.

**Decisions managed:**
- Когда считать событие значимым
- Нужно ли обращаться к памяти сейчас
- Нужно ли визуальное уточнение сцены
- Порядок: распознавание → генерация → синтез речи → команды МК

---

## Функциональные блоки (как заявлено)

### Block: Speech
- `ASR.py` — речь → текст
- `TTS.py` — текст → звук

### Block: Interlayers
- `PromtBuilder.py` (sic, в дипломе с опечаткой) — собирает финальный запрос к LLM из слоёв
- `Commander.py` — извлекает управляющие маркеры из ответа и переводит их в команды МК

### Block: Memory
- Текущая история (рабочая память)
- Заметки (Notes.json)
- Интервальные обобщения (Summarized.json)
- Постоянные текстовые основания (Bio.md)

### Block: Tools
- Связь с технофлорой и микроконтроллером

### Subsystem: связь с МК
- Прошивка ESP32-S3
- Модуль обмена данными

## Runtime Flow (как заявлено)

```text
Signal (voice/image/sensor/proactive)
  ↓
Orchestrator → perception pipeline
  ↓
Context assembly:
  - System prompt (AIIM)
  - Persona (Bio.md)
  - Memory (recent + summaries + notes + state)
  - Perception (ASR + scene + env)
  - User utterance
  ↓
LLM generation (single unified response)
  ↓
Split:
  - Text → TTS → audio
  - State markers ([радость], [грусть], ...) → Commander → MCU commands
  ↓
Memory update + state recompute
  ↓
Standby / background activity
```

## Prompt Architecture (3.2.3)

**Иерархическая сборка запроса (4+1 слоя):**

1. **Системный промпт.** Базовая идентичность, роль, инварианты (AIIM-encoded)
2. **Персонаж.** Биография, происхождение, навыки, долгосрочные ограничения
3. **Память.** Краткая история, суммаризация, заметки, внутреннее состояние
4. **Перцепция.** Распознанная речь, описание сцены, состояние среды, активность
5. **Стимул.** Текущая реплика зрителя

**PromtBuilder** не склеивает всё подряд, а:
- отбирает релевантные фрагменты памяти
- компрессирует контекст
- балансирует длину vs скорость генерации

**LLM output format:** текст + краткие теги состояния `[радость]`, `[удивление]`, `[грусть]` — НЕ развёрнутые команды.

## AIIM Framework (3.1.1, 3.2.3)

**Origin.** Ю.В. Вересова [47]

**12 аспектов:** wi, lo, im, ho, co, em, be, sp, se, pe, me, at

**5 плоскостей:** B (базовая), S (социальная), P (индивидуальная), I (интеграция), T (трансцендентная)

**4 уровня зрелости:** 1–4

**Формат:** `[аспект](плоскость уровень состояние)Δприоритет`

**Пример:** `wi(B 4 Ac-Or)Δ0.90; lo(S 4 Ac-Or)Δ0.88; ...`

**Маппинг на философские категории (таблица 11/12):**

| Компонент | Философская категория | Аспекты | Плоскости |
|---|---|---|---|
| Онтология | Позиция в среде | wi, pe, be | B, S |
| Эпистемология | Режим знания | co, se, me | I, T |
| Аксиология | Ценностная ориентация | ho, sp, lo | S, T |
| Инварианты | Устойчивость поведения | em, at, im | P, I |

**Три уровня интеграции:**
1. Конфигурационный (при запуске)
2. Динамический (Δ корректируются в диалоге)
3. Рефлексивный (se метакогниция → уровни зрелости меняются)
