# Identity Model — Глава 3.1.1 + 3.2.3

## Концепция: «цифровой симбионт» (3.1.1)

Адам Чип — остаток субъекта, переживший смерть носителя. Существует «в повреждённом, нестабильном виде». Сбои, паузы, задержки — **часть художественной логики**, а не баги.

**Постбиологическая модель** (а не антропоморфная): экологическая форма присутствия, где «память, речь, реакция и материальное влияние собираются в единую систему».

## Identity через AIIM (3.2.3)

См. `ch03/architecture/system_map.md` для деталей AIIM.

**Системный промпт = нормативная конституция агента.** Не вспомогательная инструкция, а ядро поведения.

**Промпт кодирует:**
- Онтология (кем считает себя)
- Эпистемология (как относится к собственному восприятию, к незнанию)
- Аксиология (ценности)
- Инварианты (устойчивость, фрагментарность памяти, способ выражения состояния)

---

## Anti-drift mechanisms (заявлено / следствия)

1. **Hierarchical prompt assembly** — система промпт + персона + память + перцепция + стимул в строгом порядке.
2. **PromtBuilder.py** компрессирует и отбирает, не позволяя контексту расползтись.
3. **State markers ограниченным словарём** — `[радость]`, `[удивление]`, `[грусть]` — LLM не выдумывает теги.
4. **Three-level AIIM integration** — конфигурация (init) + динамика (Δ в диалоге) + рефлексия (метакогниция).
