# Evaluation Criteria — 8 критериев квазисубъектности (раздел 2.1)

Backbone верификации Stage 2. Извлечено из главы 2 диплома (стр. 46–60).

---

## Crit 1: Степень автономизации (2.1.1)

**Определение.** Источник целеполагания: где система получает импульс к действию. Реактивная конфигурация отвечает на запрос, проактивная — продолжает процесс после завершения, автогенеративная — инициирует действия из собственного состояния.

**Операционализация.** Четыре уровня (таблица 3):

| Уровень | Источник цели | Признак |
|---|---|---|
| 1. Реактивный | Полностью пользователь/среда | Не инициирует за пределами ответа |
| 2. Контекстно-управляемый | Пользователь/среда | Распределяет промежуточные шаги |
| 3. Проактивный | Подцели, продолжение после запроса | Ограниченная самоподдержка |
| 4. Автогенеративный | История, внутреннее состояние, конфигурация среды | Удерживает и формирует задачи без внешнего запроса |

**Ожидаемые признаки в коде:**
- Voice loop, который продолжает работу после завершения turn
- Background workers (scene worker, memory consolidator)
- Session watchers / cron jobs

**Связь с разделом 3.4.** Метрика 3.4.4 (интеракционность и инициатива).

## Crit 2: Тип агентности (2.1.2)

**Определение.** Топология локализации действия: где проходит операциональный контур.

**Операционализация.** Четыре типа (таблица 4):

| Тип | Характеристика |
|---|---|
| Централизованная | Восприятие, решение, действие в одном агенте/модели |
| Модульная | Функции распределены между подсистемами, единый центр управления |
| Гибридная | Часть в едином агенте, часть в распределённой инфраструктуре |
| Распределённая | Поведение из координации нескольких агентов, не сводится к центру |

**Ожидаемые признаки в коде:**
- Орchestrator + memory + tool layer + reflection module → модульная или гибридная
- Action layer как отдельный pipeline-validator
- Persona graph отдельно от runtime graph

**Связь с разделом 3.1.2 (логика поведения), 3.2.1 (общая архитектура).**

## Crit 3: Устойчивость идентичности (2.1.3)

**Определение.** Сохраняется ли узнаваемая линия поведения между взаимодействиями. Базируется на нарративной идентичности Рикёра (ipse), перформативности Гоффмана и Батлер.

**Операционализация.** Четыре режима (таблица 5):

| Режим | Источник поведения | Характеристика |
|---|---|---|
| Отсутствующая | Нет истории | Не зависит от прошлого опыта |
| Статическая | Внешняя конфигурация | Образ задан извне, почти не меняется |
| Динамическая | Опыт, память, взаимодействие | Изменяется на основе памяти |
| Диссоциированная | Несколько внутренних состояний | Распределена между субагентами |

**Ожидаемые признаки в коде:**
- System prompt с устойчивой ролью
- Persona files (Identity.md, Lore.md, Abilities.md)
- Anti-drift mechanisms (echo gate, leading noise filter)
- Tuning store (configurable персона)

**Связь с разделом 3.2.3 (системный промпт), 3.4.2 (метрики удержания роли).**

## Crit 4: Режим нормативности (2.1.4)

**Определение.** Источник правил действия: внешний код / системные инструкции / адаптивные стратегии.

**Операционализация.** Четыре типа (таблица 6):

| Тип | Источник правил | Характеристика |
|---|---|---|
| Внешняя | Разработчик, сценарий, код | Не перерабатывает правила |
| Ограниченно-внутренняя | Заданные рамки | Адаптируется в пределах |
| Гибридная | Внешние ограничения + внутренние стратегии | Сочетание |
| Внутренняя | Собственные правила | Вырабатывает свои критерии |

**Ожидаемые признаки в коде:**
- Action layer с whitelist scenes
- Safety constraints (motor_max_duration_ms, half_duplex_mute)
- Echo gate (правила выбора реплик)
- Leading noise filter (правила очистки контекста)
- Salience rules в episodic memory

**Связь с разделом 3.4.2 (метрики нормативной устойчивости).**

---

## Crit 5: Темпоральная связность (2.1.5)

**Определение.** Связь действий во времени: от эпизодической к процессуальной. Базируется на Кантовском временном единстве, плановой теории Братмана, нарративной идентичности Рикёра.

**Операционализация.** Четыре уровня (таблица 7):

| Уровень | Горизонт | Пояснение |
|---|---|---|
| Эпизодическая | Отдельный стимул | Не образует последовательности |
| Контекстная | Одна сессия/задача | Связь в пределах сессии |
| Нарративная | История взаимодействий | Прошлое влияет на текущее |
| Процессуальная | Непрерывный цикл | Память + планирование + рефлексия + опыт |

**Ожидаемые признаки в коде:**
- Episodic memory (SQLite + JSONL)
- Session accumulator
- Salience scoring
- Memory consolidator (daily cron)
- Long-term summaries

**Связь с разделом 3.2.4 (память), 3.4.3 (метрики памяти и темпоральной связности).**

---

## Crit 6: Интеракционность (2.1.6)

**Определение.** Форма отношения с другим: собеседником, средой, группой. Базируется на драматургии Гоффмана, перформативности Батлер, акторно-сетевой теории Латура.

**Операционализация.** Четыре типа (таблица 8):

| Тип | Характер контакта |
|---|---|
| Реактивное | Отвечает на отдельные действия |
| Диалоговое | Удерживает контекст, обменивается репликами |
| Кооперативное | Совместное решение задачи |
| Координационное | Согласование с другими агентами/средой |

**Ожидаемые признаки в коде:**
- VAD (voice activity detection)
- Wake word engine (openwakeword)
- Voice loop controller с half-duplex
- Filler phrases (TTS while LLM generating)
- Reply window with absolute deadline

**Связь с разделом 3.2.5 (перцептивный/речевой контуры), 3.4.4 (метрики интеракционности).**

---

## Crit 7: Воплощённость (2.1.7)

**Определение.** Степень включённости системы в среду. Базируется на бытии-в-мире Хайдеггера, теории воплощённого познания Кларка-Чалмерса, материальности информации Хейлз.

**Операционализация.** Четыре уровня (таблица 9):

| Уровень | Характеристика системы |
|---|---|
| Символическая | Только текстовый/знаковый интерфейс |
| Мультимодальная | Голос + изображение + жест |
| Виртуально-воплощённая | Симуляция, действия меняют мир |
| Физическая | Сенсоры, моторы, роботы, физическое пространство |

**Ожидаемые признаки в коде:**
- MCU client (ESP32 motor control)
- Camera reader (MJPEG stream)
- Scene worker (VLM analysis)
- TTS audio output (HDMI ALSA)
- Action layer для физических scenes (boot_idle, all_on, alternating)

**Связь с разделом 3.3.2 (перцептивный/моторный слои), 3.3.3 (программирование МК).**

---

## Crit 8: Уровень эмерджентности (2.1.8)

**Определение.** Где возникает поведение: локальная архитектура / взаимодействие / системный эффект. Базируется на индивидуации Симондона, акторно-сетевой теории Латура, реляционной онтологии Барад.

**Операционализация.** Три уровня (таблица 10):

| Уровень | Источник поведения | Характеристики |
|---|---|---|
| Локальный | Архитектура одного агента | Определяется его внутренними состояниями |
| Интеракционный | Взаимодействие с пользователем/средой/агентами | Возникает в процессе контакта |
| Системный | Вся конфигурация | Эффект всей системы, не сводится к элементу |

**Ожидаемые признаки в коде:**
- Cross-module coordination (orchestrator + persona + memory)
- Event bus + JSONL log (события из разных модулей собираются в одну ленту)
- Emergent dialogue (LLM + scene + memory + echoes + tuning)
- Communities в коде-графе с межгрупповыми связями

**Связь с разделом 3.4.5 (интерпретация результатов и ограничения).**
