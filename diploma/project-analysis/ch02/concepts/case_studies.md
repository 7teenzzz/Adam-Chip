# Case Studies — Сравнительный анализ (2.2–2.3)

## Соло-агенты (2.2)

### 2.2.1 Agent Ruby (Линн Хершман Лисон)

**Архитектура.** Интерактивный веб-портал, эволюционировавший от чат-бота к долгосрочной цифровой личности.

**Ключевые свойства.**
- Авторская нарративная идентичность
- Веб-интерфейс как сцена (по Гоффману)
- Длительное взаимодействие через возврат пользователя

**Что заимствовано в Adam Chip.** Концепция нарративной идентичности через персона-файлы (Identity.md, Lore.md).
**Что отвергнуто.** Чисто текстовый интерфейс — Adam Chip воплощён через моторику и голос.

### 2.2.2 Being (2024)

**Архитектура.** Voice-first персональный ассистент с долговременной памятью.

**Ключевые свойства.**
- Persistent memory с retrieval
- Голосовой канал
- Личная адаптация

**Заимствовано.** Voice-loop pattern, persistent memory architecture.
**Отвергнуто.** Замкнутая на одного пользователя модель — Adam Chip публичная инсталляция.

### 2.2.3 Voyager (Wang et al., 2023)

**Архитектура.** LLM-агент в Minecraft с автоматической генерацией задач и иерархическим планированием.

**Ключевые свойства.**
- Curriculum learning через iterative prompting
- Skill library (composable code skills)
- Self-verification

**Заимствовано.** Composability of skills (для будущих расширений Adam Chip).
**Отвергнуто.** Open-ended exploration — Adam Chip имеет фиксированную exhibition-сцену.

## Мультиагентные системы (2.3)

### 2.3.1 Bag Of Beliefs (Ian Cheng)

**Архитектура.** Художественная multi-agent система с BDI (Belief-Desire-Intention) логикой.

**Ключевые свойства.**
- Распределённая идентичность
- Эмерджентное поведение из взаимодействия агентов

**Заимствовано.** BDI-инспирация для удержания intention в долговременной памяти.
**Отвергнуто.** Многоагентная топология — Adam Chip единый агент с модульной архитектурой.

### 2.3.2 Generative Agents (Park et al., 2023)

**Архитектура.** 25 LLM-агентов в симуляции, с памятью, рефлексией, планированием.

**Ключевые свойства.**
- Memory stream + retrieval (recency + importance + relevance)
- Reflection cycle (агент периодически синтезирует общие наблюдения)
- Planning hierarchically

**Заимствовано.** Salience scoring в episodic memory; reflection-style consolidator.
**Отвергнуто.** Multi-agent simulation — Adam Chip один агент.

### 2.3.3 Symbiosis of Agents (упоминается)

**Архитектура.** Художественная инсталляция с распределённой агентностью.

**Заимствовано.** Концепция связи между художественной формой и инженерной структурой.
**Отвергнуто.** Сложная межагентная координация.

## Синтез (2.4)

**Архитектурные выводы для Adam Chip:**

1. **Quasisubjectivity-as-effect.** Достаточно убедительная конфигурация модулей (память + identity + loop) воспроизводит эффект субъекта без необходимости полной автономности.

2. **Single-agent + modular composition.** Не нужна multi-agent топология — модульная сборка единого агента из persona + memory + voice loop + action layer + perception достаточна.

3. **Embodiment matters.** Voice + motor + camera повышают уровень квазисубъектности по сравнению с чистым чатом.

4. **Anti-drift critical.** Без механизмов удержания роли (echo gate, salience, system prompt) идентичность распадается за несколько turn'ов.

5. **Author as meta-architect (2.4.2).** Автор инсталляции — не программист агента, а архитектор его условий существования: задаёт ресурсы (память, персона), правила (action layer, safety), но не каждый ответ.
