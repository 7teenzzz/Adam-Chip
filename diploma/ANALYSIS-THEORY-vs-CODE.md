**Что совпадает:**

**Что НЕ совпадает — Mood-AIIM мостик:**

**Диплом (3.2.6, после таблицы Mood↔AIIM):**
> Переходы между состояниями не свободны: их выбор определяется тем же контуром Δ-сдвигов аспектов AIIM, который описан в разделе о системном промпте (3.2.3). Доминирование `lo` после нескольких эмпатичных обращений сдвигает выбор тега к `warm`; активация `ho` … переключает выбор на `irritated`.

**Код [`action.py:55–62`](../System/adam/action.py#L55):**
```python
if any(word in text for word in ("рад", "интересно", "вижу", "привет")):
    return ... mood="warm" ...
if any(word in text for word in ("нет", "осторож", "не могу", "не стоит")):
    return ... mood="irritated" ...
```

**Прямой keyword-match по русским словам в ответе LLM — без всяких AIIM-аспектов, без Δ-сдвигов.**

**Решение:** зависит от выбора в 3.2.3 (Вариант A vs B):
- Если A (реализовать AIIM) — переписать `action.py` так, чтобы mood выбирался по доминирующему аспекту в `Tuning.json::aiim`.
- Если B (теоретическая рамка) — оставить текст 3.2.6 но снять буквальную привязку к Δ-механике; описать таблицу Mood↔AIIM как «концептуальная карта соответствий, реализуемая в текущей версии через keyword matching как первое приближение».
