    ```markdown
    ## Audio Output Path (ESP32 Speaker)

    Текущий target TTS — `esp32_speaker` (Phase 29, 2026-05-18). Голос Адама воспроизводится из корпусных динамиков через цепочку Jetson → HTTP POST → ESP32 → PCM5102A I2S DAC → резистивный делитель 1:6 → PAM8403 class-D × 16 → 2 параллельных 8Ω 1W драйвера на канал (4Ω нагрузка).

    ### Hardware topology

    - **DAC:** PCM5102A на ESP32 I2S_NUM_1 (BCLK=GPIO38, LRCK=GPIO39, DATA=GPIO40, MCLK→GND). Sample rate fixed 44100 Hz mono 16-bit.
    - **Attenuator:** делитель 1:6 на каждый канал — R1=10 кОм series + R2=2 кОм to GND между PCM5102A LOUT/ROUT и PAM8403 INL/INR. Без делителя 2.1 Vrms × ×16 gain = hard clip.
    - **Amplifier:** PAM8403 class-D, BTL output, 5V supply (общая ветка с ESP).
    - **Load:** 2 параллельные 8Ω 1W колонки на канал = 4Ω. Каждая пара изолирована (BTL inviolable: `−OUT_L ≠ −OUT_R`, `−OUT_x ≠ GND`).
    - **Power supply:** общая 5V с ESP через тот же step-down модуль + локальный 100 µF + 100 nF decoupling у пинов VDD PAM8403.

    Полная BOM и обоснование решений: `.planning/phases/29-audio-out-esp32-pcm5102a-pam8403-2-8-parallel/29-CONTEXT.md`.

    ### Volume cap (defense-in-depth)

    Software-cap `tuning.voice.volume.maximum = 1.0` зафиксирован в `System/Config.schema.json` и `System/adam/tuning.py` (pydantic `le=1.0`). UI tuning slider не позволяет выставить > 1.0; API PATCH `/api/config` отклоняет. Это защита 1W драйверов от thermal failure.

    ### Failover на jetson_hdmi (procedure)

    Если динамики ESP не работают (нет звука, хрип, ESP firmware crash) — переключение обратно на HDMI Jetson:

    ```bash
    # 1. Изменить Config.json
    jq '.services.tts.output_target = "jetson_hdmi"' System/Config.json > /tmp/config-new.json && mv /tmp/config-new.json System/Config.json

    # 2. Restart orchestrator (output_target читается в TTSClient.__init__)
    sudo systemctl restart adam-orchestrator.service

    # 3. Verify
    curl --noproxy '*' -fsS http://127.0.0.1:8080/api/agent/status | jq
    # Сделать пробный turn — голос должен идти через HDMI (services.tts.output_device, обычно plughw:1,3)
    ```

    Обратное переключение — изменить значение на `"esp32_speaker"` + restart.

    ### Диагностика типичных проблем

    **Симптом: тишина в динамиках, в events.jsonl tts_finished `ok: true`**
    - Проверить ESP speaker readiness: `curl --noproxy '*' -fsS http://10.10.10.171/api/status | jq '.speaker_ready'` (ожидание `true`).
    - Проверить питание PAM8403: 5V на VDD пину.
    - Проверить физические соединения между ESP, делителем, PAM8403 и динамиками.
    - Проверить firmware boot log COM6: `bootLog("speaker", "ready ...")` должен быть.

    **Симптом: хрип / клип на длинных гласных**
    - Проверить `tuning.voice.volume`: должен быть ≤ 1.0. `jq '.tuning.voice.volume' System/Config.json`.
    - Если volume в норме — делитель собран без R2 ИЛИ R1/R2 перепутаны (R1=10kΩ ESP-сторона, R2=2kΩ GND-сторона).

    **Симптом: треск / щёлканье в такт работе моторов (PCA9685 burst)**
    - Power-rail noise coupling. Mitigation: добавить LC-фильтр между step-down и PAM8403 VDD (22 µH дроссель 0.5A + 470 µF электролит 10V). См. `29-RESEARCH.md §Integration Pitfalls #7`.

    **Симптом: self-echo — Адам слышит сам себя, в events.jsonl asr_result в окне tts**
    - Поднять `services.asr.post_tts_discard_window_ms` через PATCH /api/config (hot-reload). Default 2500; типичный bump 3000-3500.
    - Verify через snippet:
      ```bash
      jq -c 'select(.event=="tts_finished" or .event=="asr_result")' data/adam/events.jsonl | tail -50
      ```

    **Симптом: динамики горячие (> 40°C на ощупь) после длительной сессии**
    - Опустить `tuning.voice.volume` через PATCH /api/config (no restart needed). Эмпирический safe-level из Phase 29 ramp — записан в `29-05-SUMMARY.md`.
    - Если volume уже на минимуме и динамики всё равно горячие — проверить, не замкнуто ли +OUT_L/+OUT_R или нагрузка < 4Ω (отвалился ли один из двух параллельных динамиков?).

    **Симптом: PAM8403 chip горячий (> 60°C)**
    - Stop immediately, power off. Топология делителя собрана наоборот (R1=2k на ESP, R2=10k на GND = усиление вместо ослабления) ИЛИ короткое замыкание в нагрузке.
