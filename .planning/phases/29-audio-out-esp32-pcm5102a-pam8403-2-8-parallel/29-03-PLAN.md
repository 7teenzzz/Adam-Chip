<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Positive loopback — POST WAV на :81/speaker → HTTP 200 + чистый 440 Hz слышен</name>
  <what-built>
    Task 1 создал детерминированный 440 Hz / 1 sec / mono / 44100 / 16-bit smoke WAV. Hardware-тракт собран в Wave 1. Этот test — первый сквозной аудио-сигнал через всю цепочку Jetson HTTP → ESP HTTP → I2S DMA → PCM5102A → делитель 1:6 → PAM8403 → 2×8Ω parallel динамики.
  </what-built>
  <how-to-verify>
    1. **Pre-flight: ESP speaker ready check** (можно сделать без Wave 1 hardware, но повторить тут как sanity):
       ```
       curl --noproxy '*' -fsS http://10.10.10.171/api/status | jq '.speaker_ready, .speaker_buffer_fill, .speaker_underruns'
       ```
       Ожидание: `true`, `0`, `0`. Если speaker_ready=false → проверить serial log COM6 через `tools/flash_com7.ps1 -ListPorts` + monitor, искать `bootLog("speaker", ...)`. Может быть collision на I2S_NUM_1.

    2. **Positive loopback test:**
       ```
       curl -v --noproxy '*' -X POST \
         -H "Content-Type: audio/wav" \
         --data-binary @tests/fixtures/test_440hz_1s_-12dbfs.wav \
         http://10.10.10.171:81/speaker
       ```
       Записать в 29-03-SUMMARY.md:
       - HTTP status code (ожидание: 200)
       - Response body (ожидание: `{"status":"ok"}` или подобный)
       - Длительность curl (ожидание: близко к 1.0 sec из-за blocking sleep в Jetson-стороне, но curl сам не блокирует — ESP отдаёт 200 как только тело принято в ring; задержку увидим только на стороне аудио-выхода)

    3. **Слуховой тест:**
       - Должен быть слышен чистый, стабильный тон 440 Hz длительностью ~1 сек из обеих пар динамиков (mono → L=R в I2S firmware).
       - Тон должен звучать ровно — без модуляции, без треска, без хрипа.
       - **Громкость:** при `tuning.voice.volume` ещё не применяется (curl идёт напрямую на ESP, минуя Jetson volume gain). Звук будет на ослабленной делителем уровне (~0.35 Vrms на входе PAM8403 × 16 gain = ~5.6 Vpp на выходе PAM8403 = около 1.1 Vrms на динамиках = заметно слышно, но не громко).

    4. **Spectral verification (опционально, если есть телефон с записью):**
       Записать вывод динамиков, открыть в Audacity или прогнать `sox recording.wav stat`:
       - Пик на ~440 Hz должен доминировать
       - Гармоники (880, 1320 Hz) должны быть ниже -30 dBFS
       - Никакого 50/60 Hz hum

    **Pass criteria:**
    - HTTP 200
    - Чистый тон 440 Hz слышен в динамиках
    - Никакой модуляции от моторов (можно параллельно дёрнуть PCA9685 — синус должен остаться чистым)

    **Fail handling:**
    - HTTP 400 `speaker_wav_format_mismatch` → WAV сгенерирован неправильно; перепроверить `sox --info`, регенерировать
    - HTTP 503 `speaker_not_ready` → ESP I2S не поднялся; serial log COM6
    - HTTP 200, тишина → проверить (a) питание PAM8403, (b) physical connections, (c) делитель собран не наоборот (R1=10к на ESP-стороне, R2=2к на GND-стороне!)
    - HTTP 200, треск / хрип → делитель собран без R2 (получаем 2.1 Vrms на входе PAM8403 = hard clip) ИЛИ R1/R2 перепутаны (R1=2к на ESP, R2=10к на GND = усиление вместо ослабления = clip)
    - HTTP 200, модуляция от моторов → Wave 1 revisit с LC-фильтром (см. RESEARCH.md §Integration Pitfalls #7)

    Записать в 29-03-SUMMARY.md секция «Positive loopback».
  </how-to-verify>
