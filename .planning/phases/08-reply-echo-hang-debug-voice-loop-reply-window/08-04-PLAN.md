    2. Функция `read_events(path, last_minutes)` — итератор по JSON-line файлу, фильтрует по `ts` (последние N минут от текущего wallclock; если в events нет `ts`, использовать порядок строк).

    4. Функция `find_last_state_change_before(events, target_ts)` — возвращает последний voice_state_change event до target_ts (для контекста, кто был "виновником" gap).
    5. Функция `report(gaps, events, error_threshold)` — печатает читаемый report:
       - "OK: 12 heartbeat events, max gap 5.2 sec (ok ≤ 6 sec)"
       - "WARN: gap 8 sec at 00:08:55 (state=reply→standby reason=reply_silence_timeout)"
       - "ERROR: gap 90 sec at 00:09:15 (last state: reply, last reason: reply_silence_timeout) — voice_loop appears frozen"

    7. Exit code: 0 (ok), 1 (gap > error_threshold обнаружен), 2 (events.jsonl не найден или пустой).

    Никаких внешних зависимостей — только stdlib (argparse, json, time, pathlib, datetime, sys, os).

    - `OK: max gap N.N sec (ok ≤ 6 sec)` — loop работает, hang НЕ воспроизведён.
    - `WARN: gap 7-29 sec at ...` — был кратковременный stall; проверить нагрузку (CPU/IO в этот момент).

       sudo systemctl stop adam-orchestrator.service
       sudo journalctl -u adam-orchestrator.service --since "10 min ago" > /tmp/hang-journal.log
