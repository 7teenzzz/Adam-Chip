<objective>
Refactor reply mode в `_vad_loop` так, чтобы:

1. Логика accumulation/endpointing была **идентична** listening (один общий блок кода, без дубликата).
2. Reply имел **единственный** таймер — silence timer 4.0 сек (новое поле `_reply_silence_timeout_sec`). Двойного `reply_absolute_deadline_sec` больше нет.
3. Защита от бесконечной диктовки в reply работала через тот же `max_segment_ms`, что и в listening (общий блок в строках ~990-999 уже это делает — нужно только убрать раздельную ветку accumulation, чтобы reply попадал в этот блок естественным путём).
4. `_REPLY_GUARD_SEC = 0.6` оставался как defence-in-depth (per CONTEXT D-04, REQ-NO-SELF-ECHO-VAD).
5. `half_duplex_mute=true` инвариант не был тронут.

Это закрывает три REQ:
- REQ-REPLY-MATCHES-LISTENING — единый код accumulation; один таймер.
- REQ-NO-SELF-ECHO-VAD — `_REPLY_GUARD_SEC` сохранён.
