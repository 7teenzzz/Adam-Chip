# Phase 5 Context: Agent Protocol

## Team

2 разработчика × 2 Claude-аккаунта каждый = до 4 агентов. Каждый агент начинает сессию без памяти предыдущих сессий.

## Problem Statement

Текущее поведение Claude-агента на этом проекте зависит от того, насколько подробно пользователь объяснил контекст в начале сессии. Без явных инструкций агент:

- Приступает к реализации не уточнив scope
- Не предупреждает о конфликтах с активными ветками или текущей фазой
- Планирует задачи неструктурированно, без связи с ROADMAP.md
- Не различает «исследовательский вопрос» и «задание на реализацию»

## Decisions

**Основной артефакт:** `docs/AGENT-PROTOCOL.md`
- Один файл, четыре секции
- `@`-референс из root CLAUDE.md → автозагрузка в контекст каждой сессии
- Только предупреждения, не блоки — агент не останавливает работу принудительно

**Архитектурный принцип:** CLAUDE.md = lean entry point (инварианты, gotchas, quick start). AGENT-PROTOCOL.md = детальный протокол поведения. Разделение как Config.json / Config.schema.json.

**Режимы работы** — агент определяет режим по типу запроса:
- Advisor: исследовательский вопрос → 2–3 предложения, не имплементировать
- Planner: запрос на план → GSD-first
- Implementer: запрос на реализацию → сначала триггеры уточнения
- Debugger: диагностический вопрос → исследовать без планирования

**Триггеры уточнения** — конкретные условия, не «когда неуверен»:
1. Задача затрагивает Config.json → уточнить: глобально или branch-only эксперимент
2. Задача модифицирует inference.py / Orchestrator.py / prompt.py → предупредить: shared infrastructure
3. Глагол размытый («улучши», «рефактори») без критерия успеха → запросить метрику
4. Задача затрагивает >3 модулей из System/adam/ → описать подход, спросить подтверждение
5. Непонятно: изменение глобальное (main) или branch-only → уточнить явно

**Классификация гэпов:**
- Branch gap: не-main ветка, BRANCH.md отсутствует
- Phase gap: задача вне активной фазы из STATE.md
- Config gap: числовое значение хардкодится в коде вместо Config.json
- Invariant gap: задача нарушает инвариант из CLAUDE.md — явный отказ
- Stale gap: STATE.md не обновлялся >2 недель

**Протокол планирования:**
1. Проверить STATE.md — активная фаза
2. Проверить ROADMAP.md — задача уже запланирована?
3. Если задача крупная (>2 файлов) → рекомендовать `/gsd-plan-phase`
4. Если задача малая → inline GSD-формат: цель / файлы / действие / verify / done

**GSD reference:** в протоколе перечислить доступные GSD-скиллы с однострочным описанием каждого.

## Scope (что входит)

- `docs/AGENT-PROTOCOL.md` с четырьмя секциями
- Обновление root `CLAUDE.md`: `@docs/AGENT-PROTOCOL.md` в строке с референсами

## Scope (что НЕ входит)

- Автоматический запуск GSD-скиллов агентом (невозможно — user-only CLI)
- Eval-система для проверки поведения агента
- Правила для конкретных модулей (это задача Phase 4 per-directory CLAUDE.md)
- Принудительные блоки — только предупреждения
