## 1. Объединённая иерархия H1 → H4 (все 4 главы)

### ch00 — Введение (10 параграфов + 6 семантических блоков)

```
H1: Введение
  (block) Объект исследования            — Архитектурная модель цифрового субъекта
  (block) Предмет исследования           — Архитектурные принципы трансляции замысла
  (block) Цель                            — Устойчивое взаимодействие агент–зритель
  (block) Задачи                          — 6 операциональных подзадач
  (block) Методологическая основа         — Теория + прототипирование + наблюдение
  (block) Научная новизна                 — Архитектурная модель для художественного пространства
  (block) Теоретическая значимость        — Уточнение представлений о цифровой субъектности
  (block) Практическая значимость         — Применение в интерактивных инсталляциях
  (block) Структура работы                — 3 главы + заключение + источники
```

**Структурная особенность:** ch00 использует семантическое блочное секционирование вместо H2/H3 — это допустимо для введения, но усложняет переход в ch01 (где используется H2/H3).

### 2.2 ch02 раскрывает то, что заявлено в ch01

- **8 критериев квазисубъектности (2.1)** = операционализация Таблицы 1 из 1.1.5
- **Кейсы (2.2–2.3)** = эмпирическая проверка ветви 1.2 (от перцептрона к мультиагентным системам)
- **Мета-архитектор (2.4.2)** = новый концепт, отсутствующий в ch01 — нужно проверить через граф

### 2.3 ch03 имплементирует то, что концептуализировано в ch02

- **3.2.1 Архитектура** ↔ ch02 общая модель (FULL)
- **3.2.4 Память** ↔ 2.4.3 memory + actualization (4 слоя в коде vs заявка в ch02)
- **3.2.6 Commander.py** ↔ 2.4.3 self-reflection loop (CRITICAL GAP — упрощено до keyword matching)
- **3.3.4 Машина состояний** ↔ 1.2.2 когнитивный цикл (EXTENDED, более детально)
- **3.4 Метрики** ↔ нет прямой параллели в ch01/ch02 — это новый блок ch03 (спецификация, не реализация)

## 3. Общая навигация по логике диплома

```
ch00 (Введение)
    │  задачи 1–2 → теория и архитектура
    │  задачи 3–6 → реализация и апробация
    ▼
ch01 (Теория)                                  ch02 (Концептуальная архитектура)
    1.1 философский трек             ──────►   2.1 операционализация (8 критериев)
    1.2 технический трек             ──────►   2.2–2.3 кейсы (6 примеров)
    1.1.5 квазисубъектность ────────────────►  2.4 синтез
    1.2.2 когнитивный цикл                     2.4.3 self-reflection loop
                                                       │
                                                       ▼
                                              ch03 (Реализация Adam Chip)
                                                  3.2 нейроагентное приложение
                                                       (System/adam/, Config.json)
                                                  3.3 инсталляция
                                                       (Subsystem/AdamsServer/)
                                                  3.4 метрики тестирования
                                                       (спецификация, не код)
```

### Ключевые сходящиеся узлы

1. **Квазисубъектность** = центральная связка ch01.1.1.5 → ch02.2.1 → ch03.3.1.2
2. **Когнитивный цикл** (8 этапов из 1.2.2) = архитектурная спецификация для ch03 оркестратора и машины состояний
3. **Таблица 1 (8 критериев)** = принцип отбора кейсов в ch02 и проверочный лист для ch03 реализации
4. **Технофлора** = новый концепт ch03, не введённый в ch01/ch02 (потенциальный gap)

### Структурные аномалии

| № | Где | Аномалия | Severity |
|---|-----|----------|----------|
| A-01 | ch03 раздел 3.1 | H3 без H2-обёртки (3.1.1, 3.1.2, 3.1.3) | HIGH |
| A-02 | ch03 раздел 3.2.3 | AIIM-уровни оформлены как H1 вместо H4 | HIGH |
| A-03 | ch00 введение | Семантические блоки вместо H2/H3 | MEDIUM (допустимо) |
| A-04 | ch02 2.4.2 | Введение нового концепта "мета-архитектор" без подготовки в ch01 | MEDIUM |
| A-05 | ch03 3.4 | 7 метрик заявлены как реализованные, но являются спецификацией | CRITICAL |
