## Кандидаты EMERGENT (художественные элементы в коде, не упомянутые в дипломе)

1. **Idle scene "boot_idle"** (`Config.json::mcu.idle_scene`):
   - Конкретное состояние покоя — переход после загрузки
   - Диплом упоминает "бездействие" в общем, но не конкретный pattern "boot_idle"
   - EMERGENT в плане конкретики, не концепции

2. **VAD agressiveness 2** (Config.json::media.audio.webrtc_vad_aggressiveness):
   - Конкретный уровень "слышимости" среды — параметр восприятия проактивности
   - Не упомянут в дипломе как часть художественного дизайна
   - Рекомендация: если важно для эстетики — описать в 3.3.4

3. **Health-monitoring через ESP32 firmware** (Subsystem/AdamsServer/):
   - В прошивке есть мониторинг состояния микрофона (silence_threshold, clip_burst_threshold)
   - Это часть "тела" агента — само-наблюдение
   - В дипломе не упомянуто как художественный элемент

## Замечания для Wave 2 synthesis

1. **Технофлора (T-07 из Ф7) — частично RESOLVED после commit 6c84c71.** Инженерная расшифровка добавлена в ch03.3.1.2 (PCA9685, PCM5102A, PAM8403). Однако PCM5102A и PAM8403 в коде не появляются явно — это уровень железа, скрытый от логического графа. Path C (документировать асимметрию) — оставить как осознанное разделение Jetson↔ESP32.

2. **Проактивность — PARTIAL с тривиальной правкой:** добавление `Mode` enum в код. Path A. Это улучшит читаемость и отладку, не требует переписывания.

3. **Симбионт — MISSING ACCEPTED.** Пользовательское решение зафиксировано: художественная метафора не требует runtime-реализации. Не выносить в Phase 9 как проблему.

4. **Мета-архитектор — MISSING с возможностью лёгкой правки.** Структура иерархии есть в коде (Config→Tuning→Runtime), но не названа. Path B: либо мини-параграф в ch02.2.4.2 с явным соответствием, либо docstring в Orchestrator.py. Низкая стоимость, средний эффект.

5. **Cross-graph evidence:** esp32-граф содержит конкретные функции (initPca9685, applyPca9685Scene, speakerPlaybackTask), что подтверждает реализацию технофлоры на firmware уровне. Это сильный аргумент для C-path по технофлоре — реализация есть, она просто не на логическом уровне Jetson.

6. **Художественная категория самая компактная (5 терминов), но самая концептуально нагруженная.** В отличие от технических терминов, художественные требуют согласованности между несколькими слоями (Lore + Identity + код + прошивка). Все 5 терминов прошли проверку без CRITICAL.
