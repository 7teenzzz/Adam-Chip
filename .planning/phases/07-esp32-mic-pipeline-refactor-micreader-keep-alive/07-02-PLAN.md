<objective>
Create a new module `System/adam/mic_reader.py` containing the `MicReader` class — a long-lived asyncio task that owns the ESP32 :81 WAV stream lifecycle (open → drain → reconnect-on-error). MicReader is the sole producer for an `asyncio.Queue(maxsize=50)` consumed by VoiceLoopController, and the sole emitter of `audio_level` events.

Purpose: per D-04..D-12 + D-18..D-20, extract ESP32 stream management out of VoiceLoopController so the stream stays open across warmup TTS, mute windows, and voice_loop restarts. CameraReader is the architectural template.
