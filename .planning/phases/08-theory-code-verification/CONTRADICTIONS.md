## HIGH CONTRADICTED (Ф8 рекомендует path)

### Термин: Автономия (Φ-13)
**Диплом:** ch01.1.1.1 (стр. 7–8, §67) — Кант (отрицает машинную автономию) vs Симондон (утверждает индивидуацию через среду).
**Код:** `Identity.md` (wi 0.65: «я не отступаю») + `prompt.py` создают видимость волевой устойчивости, но все 12 аспектов AIIM детерминированы внешним конфигом `Tuning.json` и persona-файлами.
**Расхождение:** диплом обсуждает онтологический спор; код реализует перформативную видимость автономии, не онтологическую.
**Severity:** HIGH | **Recommended path:** **C** — задокументировать в ch03.2.3 явно: «для инсталляционного контекста достаточно перформативной автономии (Гоффман), не онтологической (Кант). Симондоновская позиция выбрана как оправдывающая рамка». Path B (попытка реализовать «настоящую» автономию) — не в scope проекта.

### Термин: Консолидация памяти (Τ-35)
**Диплом:** ch02.2.4.3 (имплицитно RAG); ch03.3.2.4 — «автоматическое суммирование эпизодов в дневник».
**Код:** в `episodic.py` есть `Episode.consolidated: bool` флаг и структура подготовлена; явного `consolidator.py` модуля нет, daily cron из `Engineering/consolidator.py` упоминается в README, но не интегрирован в пайплайн runtime.
**Расхождение:** диплом обещает работающий механизм, код — только заготовку.
**Severity:** HIGH | **Recommended path:** **B** — реализовать consolidator модуль (или интегрировать `Engineering/consolidator.py` в Orchestrator). Это **единственный** случай в Ф8, где path B оправдан как полноценная разработка, а не косметика.

## MEDIUM CONTRADICTED (Ф8 рекомендует path)

### Термин: Агентность (Φ-3)
**Диплом:** ch01.1.1.4 (§381) — «способность инициировать действие в среде» предполагает архитектурно разделённое решение (LLM генерирует маркеры → action layer интерпретирует).
**Код:** `action.infer()` упрощён до keyword matching по `reply_text`; LLM не выдаёт явные маркеры действия.
**Расхождение:** диплом подразумевает чистую архитектуру маркеров; код использует heuristic-парсинг.
**Recommended path:** **B** — на горизонте Ф9+ доработать `action.py` для парсинга явных mood-маркеров из LLM. Краткосрочно — оставить как есть, заметить в ch03.2.6.

### Термин: Аффект / Модуляция (Φ-5)
**Диплом:** ch01.1.2.4 (§1239–1496) — аффект как непрерывная модуляция в пределах доступных параметров (Брайдотти/Делёз).
**Код:** `action.py:8` — 5 дискретных mood-состояний (`Mood = Literal["neutral", "curious", "warm", "irritated", "silent"]`).
**Расхождение:** непрерывная философская категория vs дискретный enum.
**Recommended path:** **C** — документировать в ch03.2.6 упрощение: «дискретная аппроксимация непрерывного аффекта 5-state enum-ом оправдана инсталляционным контекстом; полная континуальная модуляция — out of scope».

### Термин: Метакогниция (Φ-7)
**Диплом:** ch01.1.2.4 (неявно) + ch03.3.2.3:27 — «оценка собственных когнитивных процессов».
**Код:** `episodic.py::salience_score()` + `episode.adam_state` — rule-based оценка важности.
**Расхождение:** диплом подразумевает онтологическое утверждение о самонаблюдении; код реализует функциональный эквивалент через salience scoring.
**Recommended path:** **C** — документировать как «функциональный эквивалент»; уточнить терминологию в ch01.1.2.4 (рефлексия ≠ метакогниция полная).

### Термин: Эмоциональный тег (Α-24)
**Диплом:** ch03.3.2.6:741 — термин назван, формализм не развёрнут.
**Код:** явный `Mood` enum из 5 состояний.
**Расхождение:** код точнее, чем диплом.
**Recommended path:** **A** — добавить в ch03.3.2.6 явное перечисление 5 mood-состояний с привязкой к `Mood` enum в action.py.

### Термин: Persona conditioning — механизм (Α-25)
**Диплом:** ch01.1.2.4:1388 — упомянуто как стиль модуляции, без описания hot-reload.
**Код:** `tuning.py::TuningStore` читает `Tuning.json` каждый turn без рестарта.
**Расхождение:** код описан скуднее, чем реализован.
**Recommended path:** **A** — мини-параграф в ch01.1.2.4 или ch03.3.2.3 о механизме hot-reload через `Tuning.json`.

### Термин: Контекст — 3 значения (Τ-28)
**Диплом:** ch01.1.2.1 + ch03.3.2.4 — три разных контекста: окно / слой / ситуационный.
**Код:** все три типа интегрированы в единый `_CTX_HEADER` блок в `prompt.py` без явного разделения.
**Расхождение:** концепция верна, но разделение размыто.
**Recommended path:** **C** — задокументировать в ch03.3.2.4, что в runtime три контекста сливаются в одну prompt-секцию (это намеренное упрощение для уменьшения num_ctx).

### Термин: Промпт (системный/пользовательский) (Τ-29)
**Диплом:** ch01.1.2.5, ch03.3.2.3:485 — упоминает оба типа.
**Код:** системный из `persona_paths`, пользовательский из ASR + history (зашитый `limit=8`).
**Расхождение:** **BUG-расхождение:** `Config.json::history_turns=2`, но `recent_dialogue(limit=8)`.
**Recommended path:** **A** — описать в ch03 фактическое поведение (limit=8 для prompt-истории, history_turns=2 — для другой цели). Альтернативно — path B (выровнять код по конфигу), но это меняет behavior.

### Термин: Сессионная память — 8 turn buffer (Τ-30)
**Диплом:** ch03.3.2.4:651 — «последние 8 turn-ов».
**Код:** `recent_dialogue(limit=8)` — число зашито.
**Расхождение:** код корректен, но не параметризован.
**Recommended path:** **B** — вынести `session_memory_buffer_size: 8` в Config.json (Config-First принцип).

### Термин: Эпизодическая память — decay 14d (Τ-31)
**Диплом:** ch03.3.2.4:653 — 14-дневный decay.
**Код:** логика decay есть в `episodic.py`, параметра в Config нет.
**Recommended path:** **B** — вынести `episodic_decay_days: 14` в Config.json.

### Термин: Поисковая память BM25+FAISS (Τ-33)
**Диплом:** ch03.3.2.4:657 — TF-IDF + FAISS (исправлено commit 6c84c71 → BM25 + FAISS).
**Код:** `BM25Index` готов (Okapi); FAISS — skeleton.
**Расхождение:** FAISS embeddings в roadmap, не в runtime.
**Recommended path:** **C** — в ch03 пометить FAISS как «Wave 2 of memory upgrade», не как готовый компонент. Полная реализация — отдельная фаза (Memory-upgrade branch).

### Термин: Салиентность (Τ-36)
**Диплом:** ch03.3.2.4:653 — упоминаются параметры.
**Код:** `Episode.salience: float` + rule-based функция вычисления; параметров в Config нет.
**Recommended path:** **B** — экспортировать `salience_weights` в Config.json (например, `{visitor_density: 0.3, dialogue_length: 0.4, mood_variance: 0.3}`).

### Термин: Когнитивный цикл — 8 этапов (Τ-39)
**Диплом:** ch01.1.2.2:910 табл.2 — восприятие → анализ → планирование → генерация → синтез → действие → оценка → память (8 этапов).
**Код:** фактический flow ~4–5 этапов (ASR → LLM → TTS + Action).
**Расхождение:** диплом теоретически богаче, код прагматичнее.
**Recommended path:** **C** — в ch03 явно указать «прагматическая реализация: 4-этапная композиция теоретического 8-этапного цикла; промежуточные этапы (анализ, планирование, синтез) сливаются в LLM-вызов». Path B (явная 8-этапная FSM) — out of scope.

### Термин: Технофлора (Χ-45)
**Диплом:** ch03.3.1.2:160 — после 6c84c71 описаны PCA9685, PCM5102A, PAM8403.
**Код:** PCA9685 явно в `initPca9685`/`applyPca9685Scene`; PCM5102A/PAM8403 — на уровне Device Tree / HAL, не видны в логическом графе.
**Расхождение:** асимметрия Jetson↔ESP32 — REST API, не HAL-калибровка с Jetson.
**Recommended path:** **C** — в ch03.3.1.2 добавить ремарку: «PCM5102A и PAM8403 управляются ESP32 firmware на уровне HAL; Jetson обращается к ним только через REST `/speaker` endpoint — это осознанное архитектурное разделение».

### Термин: Проактивность / 3 режима (Χ-46)
**Диплом:** ch03.3.1.2:190–196 — Диалог, Бездействие, Дремота.
**Код:** 3 режима реализованы функционально (VoiceLoopController, idle_scene, SceneWorker), но без явного enum.
**Recommended path:** **A** — добавить `Mode = Literal["dialogue", "idle", "dormancy"]` enum в Orchestrator.py для явной типизации; затем сослаться на enum из ch03.3.1.2. Тривиальная правка.

### Термин: Мета-архитектор (Χ-48)
**Диплом:** ch02.2.4.2:1312–1370 — иерархия macro/meso/micro авторства.
**Код:** структура есть (`Config.json` = macro, `Tuning.json + Identity.md` = meso, `action.py + SceneWorker` = micro), но не названа «мета-архитектором».
**Recommended path:** **B** — добавить module-level docstring в Orchestrator.py с явным описанием трёх уровней авторской иерархии; альтернативно (или дополнительно) мини-параграф в ch02.2.4.2 с явным соответствием. Низкая стоимость, средний эффект.
