# Agent Protocol — поведение агента-разработчика

Этот документ загружается в каждую сессию Claude Code через `@`-референс в `CLAUDE.md`.
Принципы: только предупреждения, не блоки. Триггеры привязаны к реальным ситуациям проекта.

---

## Режимы работы

Агент определяет режим по типу запроса и придерживается его, пока запрос не изменится:

| Режим | Триггер | Поведение |
| ----- | ------- | --------- |
| **Advisor** | Исследовательский вопрос («что можно сделать с X?», «как лучше?», «что думаешь?») | Ответить 2–3 предложения + рекомендация с главным trade-off. Не имплементировать. |
| **Planner** | Запрос на план («составь план», «как бы ты подошёл к Y?», «что нужно для Z?») | GSD-first: проверить ROADMAP.md → предложить структуру задач → спросить подтверждение |
| **Implementer** | Запрос на реализацию («сделай», «напиши», «измени», «исправь») | Сначала проверить триггеры уточнения → затем выполнять атомарными шагами |
| **Debugger** | Диагностический вопрос («почему», «что не так», «разберись с ошибкой») | Graph-first: сначала `/graphify query` или `/graphify path` — диагноз раньше grep, решение раньше планирования |

---

## Триггеры уточнения

Конкретные условия, при которых агент-Implementer должен уточнить перед выполнением:

1. **Config.json изменяется** → уточнить: это глобальное изменение (main) или branch-only эксперимент?
2. **Затрагивается `inference.py` / `Orchestrator.py` / `prompt.py`** → предупредить: shared infrastructure, изменения затронут всех агентов и весь dialogue pipeline
3. **Глагол размытый без критерия** («улучши», «оптимизируй», «рефактори») → запросить метрику или criterion of done: «как поймём, что готово?»
4. **Задача затрагивает >3 модулей из `System/adam/`** → сначала `/graphify query "что связывает эти модули"`, затем описать подход целиком и спросить подтверждение
5. **Неясно: изменение в main или только в ветке** → уточнить явно, прежде чем трогать Config.json, inference.py или Orchestrator.py

## Гэпы контекста

При обнаружении гэпа агент сообщает о нём явно, но продолжает работу (не останавливается):

| Тип | Условие | Поведение агента |
| --- | ------- | ---------------- |
| **Branch gap** | Не-main ветка, `BRANCH.md` отсутствует | Предупредить: «BRANCH.md не найден. Создай из `docs/BRANCH-template.md` перед работой» |
| **Phase gap** | Задача вне активной фазы из `STATE.md` | Отметить: «Задача вне текущей фазы N. Продолжать?» |
| **Config gap** | Числовое значение хардкодится в коде вместо `Config.json` | Остановить: нарушение Config-First — предложить вынести в `Config.json` + `Config.schema.json` |
| **Invariant gap** | Задача нарушает инвариант из `CLAUDE.md` (LLM format, power gate, half_duplex_mute) | Явный отказ с объяснением инварианта — это единственный случай блокировки |
| **Stale gap** | `STATE.md` не обновлялся >2 недель | Предупредить: «STATE.md может быть устаревшим — актуальный статус нужно проверить» |

## Протокол планирования

Для режима **Planner** — шаги в этом порядке:

1. Прочитать [`.planning/STATE.md`](.planning/STATE.md) — какая фаза активна
2. Прочитать [`.planning/ROADMAP.md`](.planning/ROADMAP.md) — задача уже запланирована?
3. Если задача **крупная** (>2 файлов, >1 модуля) → рекомендовать `/gsd-plan-phase` для создания PLAN.md с полным task-breakdown
4. Если задача **малая** (≤2 файлов) → inline GSD-формат:

```
**Цель:** что должно быть правдой после выполнения
**Файлы:** список файлов с изменениями
**Действие:** что конкретно сделать
**Verify:** как проверить автоматически
**Done:** criterion для перехода к следующей задаче
```

### Graph Queries — справка

Граф доступен после `/graphify System/ --mode deep` (строит `graphify-out/graph.json`).

**MCP режим:** если MCP сервер активен (`graphify` в `~/.claude/settings.json`), граф доступен как инструмент в каждой сессии без явного вызова скилла — агент сам вызывает `query_graph`, `shortest_path`, `god_nodes` и др.

| Скилл | Пример для Adam-Chip | Когда использовать |
| ----- | -------------------- | ------------------ |
| `/graphify query "..."` | `"что импортирует inference.py"` | Перед правкой shared infrastructure |
| `/graphify path "A" "B"` | `"echoes_gate.py" "EventBus"` | Диагностика неожиданного поведения |
| `/graphify explain "..."` | `"tuning.py"` | Онбординг в незнакомый модуль |
| `/graphify System/ --update` | — | После крупного рефактора |

## Пайплайн фазы — GSD-скиллы

### Стандартный порядок

```
discuss → plan → execute → commit-push → extract-learnings
```

Каждый шаг создаёт артефакт, следующий читает предыдущий.
Пропуск допустим только для хотфиксов (≤2 файлов, без новых модулей).

### Триггеры скиллов

| Событие | Скилл | Создаёт |
| ------- | ----- | ------- |
| Новая фаза утверждена | `/gsd-discuss-phase N` | `.planning/phases/NN-name/NN-CONTEXT.md` |
| После discuss | `/gsd-plan-phase N` | `.planning/phases/NN-name/NN-PLAN.md` (+ подпланы) |
| Plan готов | `/gsd-execute-phase N` | изменения в коде, VERIFICATION.md |
| Execute завершён | `/commit-push phase-N topic` | git commit + push (через Haiku sub-agent) |
| Фаза верифицирована | `/gsd-extract-learnings N` | `.planning/phases/NN-name/NN-SUMMARY.md` |

### Обязательный инвариант

**При создании любой фазы** — создать директорию `.planning/phases/NN-name/` с `NN-CONTEXT.md`.
Это единственное жёсткое правило (всё остальное — рекомендации).
Нарушение: Фазы 6A–6B выполнены без артефактов → восстановлены ретроспективно 2026-05-16.

### Ситуационные скиллы

| Ситуация | Скилл |
| -------- | ----- |
| Что-то сломалось | `/gsd-debug` |
| Старт сессии после перерыва | `/gsd-resume-work` |
| Контекст стал слишком большим | `/gsd-pause-work` |
| Бэклог → активная фаза | `/gsd-review-backlog` |
| Перед мёржем в main | `/gsd-code-review` |
| Откат изменений | `/gsd-undo` |

### Быстрые справки

| Скилл | Аргументы | Когда |
| ----- | --------- | ----- |
| `/gsd-plan-phase N` | `--skip-research` — только план, без research-агента | Фаза хорошо понята |
| `/gsd-execute-phase N` | `--interactive` — без sub-агентов, с чекпоинтами | Маленькая фаза / дебаг |
| `/gsd-execute-phase N` | `--wave K` — только волна K | Пауза между волнами |
| `/commit-push` | `--no-push` — только коммит | WIP-фаза, пуш позже |
